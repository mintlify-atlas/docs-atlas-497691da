## Prerequisites

- lazygit installed — see the [installation guide](/installation)
- git 2.32.0 or later
- An existing git repository (or any directory where you want to initialize one)

## Launch lazygit

<Steps>
  <Step title="Open a terminal in your git repository">
    Navigate to any git repository in your terminal:

    ```sh
    cd /path/to/your/repo
    ```
  </Step>

  <Step title="Run lazygit">
    ```sh
    lazygit
    ```

    lazygit opens a full-screen terminal UI. If you run it outside a git repository, it will prompt you to initialize one with `git init`.
  </Step>
</Steps>

## The interface layout

lazygit divides the screen into five panels on the left and a main diff/detail view on the right.

| Panel | Key | What it shows |
|-------|-----|---------------|
| **Status** | `1` | Current branch, upstream tracking info, and recent repos |
| **Files** | `2` | Working tree — unstaged and staged changes |
| **Branches** | `3` | Local branches, remotes, tags, and GitHub PRs (if `gh` is installed) |
| **Commits** | `4` | Commit log for the current branch |
| **Stash** | `5` | Stash entries |

The **main panel** on the right shows a diff, file list, or detail view depending on what you have selected on the left.

<Tip>
  Press `+` and `_` to cycle through screen modes and give more space to the main panel. This also reveals the full commit graph.
</Tip>

## Navigate the interface

<Steps>
  <Step title="Switch between panels">
    Press the number key for the panel you want (`1`–`5`), or use the left/right arrow keys to move between panels.
  </Step>

  <Step title="Move within a panel">
    Use the up/down arrow keys (or `j`/`k` in vim-style) to select items within the active panel.
  </Step>

  <Step title="Dive into an item">
    Press `enter` to expand the selected item. For example:

    - In the **Files** panel — opens the file's diff line by line
    - In the **Commits** panel — shows the files changed in that commit
    - In the **Branches** panel — shows that branch's commit log
  </Step>

  <Step title="Go back">
    Press `escape` to return to the previous view.
  </Step>
</Steps>

## Your first actions

### View a diff

Select the **Files** panel (`2`). Highlight any changed file — the diff appears immediately in the main panel on the right. Press `enter` to focus the diff view and scroll through individual lines.

### Stage a file

With a file selected in the **Files** panel, press `space` to toggle it between staged and unstaged. The file moves between the unstaged and staged sections.

To stage individual lines instead of the whole file:

<Steps>
  <Step title="Enter the diff view">
    Press `enter` on a file in the Files panel to open the line-level diff.
  </Step>
  <Step title="Stage a line">
    Move to a line and press `space` to stage it.
  </Step>
  <Step title="Stage a range">
    Press `v` to start selecting a range of lines, then press `space` to stage the selection. Press `a` to select and stage the entire current hunk at once.
  </Step>
</Steps>

### Make a commit

<Steps>
  <Step title="Stage your changes">
    Stage the files or lines you want to include (see above).
  </Step>
  <Step title="Open the commit dialog">
    Press `c` in the **Files** panel to open the commit message input.
  </Step>
  <Step title="Type your commit message and confirm">
    Enter your message and press `enter` to commit.
  </Step>
</Steps>

<Note>
  Press `C` (uppercase C) to commit using your `$GIT_EDITOR` instead of the built-in input — useful for multi-line commit messages with a body.
</Note>

## Open the keybindings menu

Press `?` at any time to open the keybindings reference for the currently active panel. Every available action, its key, and a brief description are listed there. This is the fastest way to discover what lazygit can do.

## Set up the `lg` alias

Typing `lazygit` every time gets old quickly. Add an alias to your shell config:

```sh
echo "alias lg='lazygit'" >> ~/.zshrc
source ~/.zshrc
```

Replace `~/.zshrc` with `~/.bashrc` or your shell's equivalent. After that, `lg` launches lazygit.

## Change directory on exit

When you navigate between repos inside lazygit, your terminal's working directory does not change automatically. To fix that, add the following shell function to your `~/.zshrc` (or equivalent):

```sh
lg()
{
    export LAZYGIT_NEW_DIR_FILE=~/.lazygit/newdir

    lazygit "$@"

    if [ -f $LAZYGIT_NEW_DIR_FILE ]; then
            cd "$(cat $LAZYGIT_NEW_DIR_FILE)"
            rm -f $LAZYGIT_NEW_DIR_FILE > /dev/null
    fi
}
```

Then reload your shell config:

```sh
source ~/.zshrc
```

From now on, calling `lg` instead of `lazygit` will change your terminal directory to wherever you were when you exited lazygit. To exit without changing directory, press `shift+Q` instead of `q`.

## CLI flags reference

lazygit accepts the following flags and arguments:

### Starting in a specific context

```sh
lazygit [status|branch|log|stash]
```

Pass a positional argument to open lazygit focused on a specific panel:

| Argument | Panel opened |
|----------|-------------|
| `status` | Status panel |
| `branch` | Branches panel |
| `log` | Commits panel |
| `stash` | Stash panel |

### Flags

| Flag | Short | Description |
|------|-------|-------------|
| `--path <path>` | `-p` | Open lazygit in the given git repo path. Equivalent to `--work-tree=<path> --git-dir=<path>/.git/`. Incompatible with `--work-tree` and `--git-dir`. |
| `--filter <path>` | `-f` | Filter commits, reflog, and stash to those touching `<path>`. Some operations are restricted in filter mode. |
| `--work-tree <path>` | `-w` | Equivalent of git's `--work-tree` argument. Defaults to `$GIT_WORK_TREE`. |
| `--git-dir <path>` | `-g` | Equivalent of git's `--git-dir` argument. Defaults to `$GIT_DIR`. |
| `--use-config-file <paths>` | `--ucf` | Comma-separated list of config file paths to use instead of the default. Sets `$LG_CONFIG_FILE`. |
| `--use-config-dir <path>` | `--ucd` | Override the default config directory. Sets `$CONFIG_DIR`. |
| `--screen-mode <mode>` | `--sm` | Initial screen mode for the focused panel: `normal` (default), `half`, or `full`. |
| `--version` | `-v` | Print version, build date, commit, OS, and arch information. |
| `--config` | `-c` | Print the default config YAML to stdout. |
| `--print-config-dir` | `--cd` | Print the config directory path. |
| `--debug` | `-d` | Enable debug mode with logging. Use `LOG_LEVEL` env var (`debug`/`info`/`warn`/`error`) to control verbosity. |
| `--logs` | `-l` | Tail the lazygit log file (use in a separate terminal alongside `--debug`). |

### Debug workflow

Run two terminal tabs side by side:

```sh
# Tab 1 — run lazygit with debug logging
lazygit --debug

# Tab 2 — stream the log output
lazygit --logs
```

Set log verbosity with the `LOG_LEVEL` env var:

```sh
LOG_LEVEL=warn lazygit --debug
```
