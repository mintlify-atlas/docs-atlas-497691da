
    ### Debian 12 / Ubuntu 25.04 and earlier

    For older Debian and Ubuntu releases, install from the GitHub releases directly:

    ```sh
    LAZYGIT_VERSION=$(curl -s "https://api.github.com/repos/jesseduffield/lazygit/releases/latest" | \grep -Po '"tag_name": *"v\K[^"]*')
    LAZYGIT_ARCH=$(uname -m | sed -e 's/aarch64/arm64/')
    curl -Lo lazygit.tar.gz "https://github.com/jesseduffield/lazygit/releases/download/v${LAZYGIT_VERSION}/lazygit_${LAZYGIT_VERSION}_Linux_${LAZYGIT_ARCH}.tar.gz"
    tar xf lazygit.tar.gz lazygit
    sudo install lazygit -D -t /usr/local/bin/
    ```

    Verify the installation:

    ```sh
    lazygit --version
    ```

    ### Fedora / RHEL derivatives (Terra)

    An alternative to the Copr repo is the [Terra repository](https://terra.fyralabs.com/):

    ```sh
    sudo dnf install --nogpgcheck --repofrompath 'terra,https://repos.fyralabs.com/terra$releasever' terra-release
    sudo dnf install lazygit
    ```

    Terra also provides `lazygit-doc`, which contains the full lazygit docs folder.

    ### Arch Linux (AUR — development build)

    The AUR package tracks the latest commit rather than the latest release:

    ```sh
    # Install using your preferred AUR helper, e.g. yay
    yay -S lazygit-git
    ```

    See the [Arch User Repository wiki](https://wiki.archlinux.org/index.php/Arch_User_Repository) for setup instructions.

    ### NixOS — official flake

    Run the latest development version directly from the repository without installing:

    ```sh
    nix run github:jesseduffield/lazygit
    ```

    Build from source using the flake:

    ```sh
    nix build github:jesseduffield/lazygit
    ```

    To integrate lazygit into another flake-based project, use the provided overlay:

    ```nix
    {
      inputs.lazygit.url = "github:jesseduffield/lazygit";

      outputs = { self, nixpkgs, lazygit }: {
        nixpkgs.overlays = [ lazygit.overlays.default ];
      };
    }
    ```

## Binary releases

Pre-built binaries for Windows, macOS (10.12+), and Linux are available on the [GitHub Releases page](https://github.com/jesseduffield/lazygit/releases). Download the archive for your OS and architecture, extract it, and move the `lazygit` binary to a directory on your `$PATH`.

## Go install

If you have Go installed, you can install lazygit directly with:

```sh
go install github.com/jesseduffield/lazygit@latest
```

<Warning>
  If the `lazygit` command is not found after installation, make sure `~/go/bin` is on your `$PATH` (macOS/Linux) or `%HOME%\go\bin` (Windows). This is different from `C:\Go\bin`, which is Go's own binary directory.
</Warning>

## Build from source

You'll need [Go installed](https://golang.org/doc/install) to build from source.

```sh
git clone https://github.com/jesseduffield/lazygit.git
cd lazygit
go install
```

To compile and run without installing:

```sh
go run main.go
```

## Dev container feature

If you use GitHub Codespaces or a Dev Container, you can add lazygit via the third-party [dev container feature](https://github.com/GeorgOfenbeck/features/tree/main/src/lazygit-linuxbinary) based on the binary releases.

## Verify your installation

After installing, confirm lazygit is available in your `$PATH`:

```sh
lazygit --version
```

You're ready to go. See the [quickstart guide](/quickstart) to launch lazygit and take your first steps.
