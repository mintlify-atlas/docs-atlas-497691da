lazygit is an open-source terminal UI for git that lets you stage files, manage branches, rebase interactively, cherry-pick commits, and much more — all without memorizing dozens of git commands. With 75,000+ GitHub stars, it has become the go-to tool for developers who want the power of git without the friction.

## Why lazygit?

Git is powerful — but the command-line interface can be painful. Interactive rebasing forces you to edit a TODO file in your editor. Staging part of a file means stepping through hunks with a command-line prompt or hand-editing patch files. lazygit eliminates all of that friction.

## Key features at a glance

- **Interactive staging** — Stage individual lines, hunks, or entire files with a keystroke
- **Interactive rebase** — Squash, fixup, drop, reorder, and edit commits visually
- **Cherry-pick** — Copy commits across branches with `shift+c` / `shift+v`
- **Undo/Redo** — Step back through any git action using the reflog (`z` / `shift+z`)
- **Worktrees** — Manage multiple working trees for parallel branch work
- **Custom patches** — Build patches from old commits and apply them elsewhere
- **Stacked branches** — Full support for `rebase.updateRefs` stacked branch workflows
- **Custom commands** — Define your own keybindings backed by shell commands
- **Custom pagers** — Integrate delta, diff-so-fancy, ydiff, and more
- **GitHub PR integration** — See PR status for branches directly in lazygit (requires `gh`)

## Getting started

<Steps>
  <Step title="Install lazygit">
    Install via Homebrew, your distro's package manager, or download a binary release. See the [installation guide](/installation).
  </Step>
  <Step title="Open a git repository">
    Navigate to any git repository in your terminal and run `lazygit`.
  </Step>
  <Step title="Explore the interface">
    The left panel shows status, files, branches, commits, and stash. The right panel shows diffs. Press `?` at any time to view keybindings.
  </Step>
  <Step title="Customize to your workflow">
    Configure keybindings, add custom commands, and set up your preferred pager. See the [configuration guide](/configuration/overview).
  </Step>
</Steps>
